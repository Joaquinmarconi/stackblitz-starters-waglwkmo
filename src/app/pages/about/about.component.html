Acerca de Mi Aplicación de Tareas

Lista de tareas que desarrollé para el Trabajo Práctico Final del Seminario de Angular 2025.

El objetivo era aplicar los conceptos fundamentales del framework para construir una Single Page Application (SPA) funcional.

Funcionalidades y Conceptos Implementados:

  • Ruteo: La aplicación tiene dos rutas, '/tareas' y '/acerca-de', que cargan componentes diferentes.
  • Componentes: La lista de tareas usa un componente padre ('TaskList') que se comunica con componentes hijos ('TaskItem') mediante @Input y @Output para pasar datos y eventos.
  • Tipado: Se utilizó una Interfaz ('Task') para asegurar la consistencia de los datos en toda la aplicación.
  • Control de Flujo: Las plantillas usan @for para renderizar la lista de tareas y @if para mostrar contenido condicional.
  • Consumo de API: Se creó un servicio ('TaskService') que utiliza 'HttpClient' para realizar peticiones GET, POST y DELETE a una API externa de MockAPI.
  • Formularios Reactivos: El formulario para añadir tareas es un Formulario Reactivo (`Reactive Form`) que incluye validaciones de campos (requerido y longitud mínima).
